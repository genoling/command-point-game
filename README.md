## 🎮 项目简介

「博弈论：律政与代码」是一款基于 **Firebase Realtime Database** 的双人实时对战博弈网页游戏。两名玩家分别扮演 **律师** 和 **程序员**，通过打赌、贷款、下达命令等方式积累/消耗 CP（Command Point 命令点数），配合专属技能与 All-In 豪赌机制，进行线下趣味惩罚博弈。

**无需安装任何 App**，只要打开浏览器，输入相同房间号即可跨设备实时对战。

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| ⚡ **实时同步** | 基于 Firebase 实时数据库，毫秒级同步双方操作 |
| 🏠 **房间机制** | 支持自定义房间号或随机房间，多房间互不干扰 |

| 🎯 **三级指令** | 1/2/3 级命令，消耗 1/2/4 CP，梯度惩罚 |
| 🏦 **银行贷款** | 用自罚待办抵押换取 CP，对方确认完成后销账 |
| 🔥 **All In 通吃** | 孤注一掷，全部押上，对方同意则掠夺其全部 CP |
| 🛡️ **专属技能** | 律师函警告 / 致命 Bug，24 小时冷却 |
| ↩️ **撤销回滚** | 每局 3 次全局撤销机会，后悔药机制 |
| 📦 **导入导出** | 数据加密导出，支持跨版本迁移与备份 |
| 🌙 **离线沙箱** | Firebase 连接失败时自动降级为本地调试模式 |

## 📖 游戏规则

### 基础设定

- **双角色**：律师 👨‍⚖️ / 程序员 👨‍💻
- **CP (Command Point)**：命令点数，打赌获胜获得，下达命令消耗
- **命令分级**：
  - 1级：消耗 1 CP
  - 2级：消耗 2 CP
  - 3级：消耗 4 CP

### 核心玩法

#### 1. 打赌赢 CP
双方线下打赌，赢家点击「赢打赌」按钮，输入打赌内容，获得 +1 CP。

#### 2. 下达命令
选择等级 → 输入惩罚/指示内容 → 点击「下达」，从自身 CP 中扣除对应点数，命令进入对方待办列表。

#### 3. 银行贷款 🏦
CP 不足时可向银行贷款，等级对应获得 1/2/4 CP，但必须同时生成一条**自己要完成的自罚待办**作为抵押，由对方确认你在现实中完成后才可销账。

#### 4. All In 通吃 🔥
消耗自身全部 CP，向对方发起通吃挑战。对方若同意，则你的 CP 归零，对方获得你全部 CP 数量（反向掠夺机制，用于翻盘或增加游戏刺激性）。

### 专属技能（24 小时冷却）

| 角色 | 技能名称 | 效果 |
|------|----------|------|
| 👨‍⚖️ 律师 | **律师函警告** | 撤销对方最后下达的 1 条命令，全额退还其消耗 CP |
| 👨‍💻 程序员 | **致命 Bug** | 扣除对方当前 30% CP（向上取整，最少扣 1 点） |

### 安全机制

- **角色密码**：每个角色可独立设置操作密码，设定后消耗 CP / 发动技能均需校验，一次生效后置灰锁定
- **管理员密码**：全局管理权限，可重置房间、管理角色密码、批量删除房间

## 🏗️ 项目架构

```
command-point-game/
├── index.html      # 主入口 · 页面结构与模板
├── style.css       # 样式层 · 主题与组件样式
├── config.js       # 配置层 · Firebase 初始化与连接看门狗
├── app.js          # 逻辑层 · 状态机、规则引擎、安全中间件
└── README.md       # 文档
```

### 设计原则

- **关注点分离**：结构 / 样式 / 配置 / 逻辑四层解耦
- **防御式编程**：Firebase 初始化失败自动降级，不阻塞 UI
- **状态驱动**：单一数据源（currentDbState），统一渲染入口
- **安全中间件**：所有敏感操作经过 secureAction 守卫
- **可追溯性**：战斗日志 + 撤销历史快照，操作留痕

## 📁 文件说明

### index.html
页面主模板，包含所有 UI 结构：顶部导航、规则面板、角色卡片、操作区、战斗日志、以及 9 个功能弹窗（打赌/贷款/管理员登录/管理面板/密码校验/房间设置/导出/导入/AllIn确认）。

### style.css
全局样式表，定义了：
- 自定义滚动条样式
- 角色卡片选中高亮（替代原巨型水印方案）
- 按钮点击微反馈
- 过渡动画规范

### config.js
Firebase 配置与初始化模块：
- 封装 firebaseConfig 配置对象
- 带 try-catch 防护的初始化看门狗
- 连接失败时 db 置空，触发 app.js 自动降级

### app.js
核心逻辑引擎（约 600 行），分为：
1. **状态管理层** - currentDbState 全局快照
2. **渲染引擎** - renderUI() 统一重绘
3. **安全中间件** - secureAction() 角色权限与密码校验
4. **规则引擎** - 打赌/贷款/指令/技能/AllIn 全部游戏规则
5. **撤销系统** - 3 步历史快照回滚机制
6. **管理员模块** - 房间管理与密码覆盖控制
7. **导入导出** - Base64 加密数据迁移

## 🌐 部署指南

### Firebase 配置替换

编辑 `config.js` 中的 `firebaseConfig` 对象，替换为你自己的 Firebase 项目配置：

```javascript
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT.firebaseapp.com",
    databaseURL: "https://YOUR_DB.firebaseio.com",
    projectId: "YOUR_PROJECT_ID",
    // ...
};
```

### Firebase 规则设置

在 Firebase 控制台 → Realtime Database → 规则中，设置为：

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

> ⚠️ 生产环境建议配合 Firebase Auth 添加更严格的安全规则。

### 默认管理员密码

初始管理员密码：`8868`

进入游戏后，点击底部「管理设置」→ 验证密码 → 进入后台可修改。
